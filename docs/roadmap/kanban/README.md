# 市场价格与供应质量看板 PRD

版本：v0.1.0
日期：2026-07-04
状态：P0 已实现，P1/P2/P3 关键事件闭环已部分落地；缓存效率和供应质量已支持从真实 usage/错误日志自动派生，公开价格 URL 正文采集与验收证据自动回填已落地，真实路由执行仍在后续阶段
范围：同行售价分析、自有号池质量控制、第三方供应商接入验收、真实成本核算、定价建议和运营看板。

## 1. 背景

当前运营侧出现两个明确诉求：

1. 分析同行的售价，知道市场上同类 API 中转、号池、模型调用服务的价格区间。
2. 控制自己的质量，避免只追低价而牺牲模型真实性、可用率、延迟、账单可信度和用户体验。

结合项目现状，Admin Plus 已经具备供应商台账、倍率检测、用量成本、健康探针、纯度检测、渠道索引采集和站点目录等基础能力。因此本需求不应做成孤立的人工表格，而应成为运营决策看板：

- 对外看市场：同行卖多少钱、以什么口径卖、近期是否降价或促销。
- 对内看供给：自有号池和第三方供应商的真实成本、稳定性、模型纯度、余额和风险。
- 对业务看结果：我们是否能以可持续毛利提供相同模型，哪些供应源可用于生产，哪些只能观察或下线。

号池轮询还存在一个容易被低估的致命问题：同一用户、同一项目或同一长上下文请求如果被轮询到不同上游账号，会破坏 provider 侧 prompt cache / context cache 的命中率。结果是账面单价看似便宜，但实际缓存读命中低、输入 token 重复计费、TTFT 变慢、长上下文成本升高。因此缓存命中率和路由亲和性也必须进入竞争分析维度。

## 2. 设计结论

1. 第一阶段做内部运营看板，不直接改线上售价，不自动切换线上流量。
2. 同行售价只采集公开可访问信息，不绕过登录、风控或权限边界。
3. 自有号池和第三方供应商都进入统一决策视图，但保留不同成本口径。
4. 价格建议以真实成本、风险缓冲和目标毛利为主，同行售价只作为市场边界参考。
5. 质量优先级高于低价，低价但模型不纯、错误率高、限流严重或账单不可信的供应源不得进入主路由。
6. 第三方供应商接入前必须完成连通性、模型列表、纯度检测、轻量调用和 usage 计量核对。
7. 自有号池也必须纳入质量治理，不能因为来源可控就跳过余额、额度、限速、并发和健康检测。
8. 轮询号池必须单独评估缓存效率；缺少用户/会话/项目级 sticky routing 的供应源，应在质量评分和真实成本中扣分。

## 3. 核心目标

### 3.1 市场价格情报

- 采集同行站点、套餐页、倍率说明、模型价格、充值赠送和最低充值门槛。
- 为每个模型沉淀市场低价、中位价、高价和近期价格变化。
- 识别异常低价，提示可能存在低纯度模型、亏本促销、限量套餐、虚假宣传或附加限制。

### 3.2 真实成本核算

- 自有号池成本来自采购成本、订阅成本、额度成本、实际消耗、账号损耗和人工调整。
- 第三方供应商成本来自供应商售价、实际倍率、充值赠送、账单消耗、余额快照和汇率/手续费。
- 所有供给最终归一为模型维度的真实单位成本，用于计算毛利和定价建议。

### 3.3 供应质量控制

- 对每个供应来源计算质量分，包括可用率、错误率、TTFT、总耗时、限流次数、并发容量、余额风险、模型身份一致性和 usage 可信度。
- 质量结论分为：`production`、`watching`、`low_priority`、`paused`、`blocked`。
- 任何供应来源在进入生产前必须有可解释的质量报告。

### 3.4 定价建议

- 对每个模型生成三类价格建议：
  - 保底价：覆盖真实成本、支付手续费、坏账风险和基础毛利。
  - 建议价：对齐市场中位数，并保留目标毛利。
  - 进攻价：低于主要同行，但需要限制额度、模型范围或质量等级。
- 当市场售价低于我们的可持续成本时，系统应提示“不建议跟价”。

### 3.5 缓存效率与路由亲和

- 评估同行和自身供应源是否支持用户级、会话级、项目级或 API Key 级路由亲和。
- 记录 OpenAI `cached_tokens`、Claude `cache_read_input_tokens` / `cache_creation_input_tokens`、Gemini `cachedContentTokenCount` 等可观测缓存信号。
- 对轮询号池计算缓存惩罚成本，把低缓存命中导致的重复输入 token、延迟上升和上下文重放成本纳入真实成本。
- 将缓存命中率作为竞争分析维度，区分“低标价但缓存差”和“标价略高但长上下文实际成本更低”的供应源。

## 7. 核心流程

### 7.1 同行售价分析

```text
采集或录入同行价格来源
  -> 解析站点、模型、价格、倍率、套餐、促销和来源 URL
  -> 归一化模型名称和计费口径
  -> 写入价格快照
  -> 与历史快照对比
  -> 生成价格变化事件
  -> 汇总市场低价、中位价和异常低价
```

### 7.2 自有号池质量控制

```text
导入或读取自有号池账号
  -> 采集余额、额度、限速、并发和模型列表
  -> 执行健康探针、纯度检测和缓存命中审计
  -> 汇总使用成本、账号损耗和轮询导致的缓存惩罚成本
  -> 生成质量报告
  -> 标记 production / watching / low_priority / paused / blocked
```

### 7.3 第三方供应商接入验收

```text
从站点目录或手工创建供应商
  -> 校验 Base URL、后台地址和账户状态
  -> 拉取模型列表和公开倍率
  -> 执行轻量真实调用
  -> 执行纯度检测、usage 计量核对和缓存命中审计
  -> 同步余额和账单快照
  -> 生成接入验收报告
  -> 通过后才允许标记为生产候选
```

### 7.4 定价建议

```text
读取市场价格快照
  -> 读取自有号池和第三方供应商真实成本
  -> 读取缓存效率、质量分和风险等级
  -> 按模型计算保底价、建议价、进攻价
  -> 计算预期毛利率
  -> 当市场价低于可持续成本时标记不建议跟价
```

### 7.5 缓存命中审计

```text
按用户/项目/会话构造多轮长上下文样本
  -> 分别测试固定账号、普通轮询和 sticky routing 策略
  -> 读取 provider 返回的缓存读写 token 和 usage 明细
  -> 计算缓存命中率、重复输入 token、TTFT 差异和实际成本差异
  -> 将低缓存命中供应源标记为成本风险或质量风险
```

## 8. 质量评分建议

质量分建议先采用简单加权，避免过早复杂化：

| 指标 | 建议权重 | 说明 |
|------|----------|------|
| 可用率 | 20% | 最近 1h / 24h 探针成功率 |
| 错误率 | 15% | 真实调用错误和上游错误 |
| 延迟 | 15% | TTFT 和总耗时 |
| 缓存效率与路由亲和 | 15% | 缓存命中率、缓存读 token、sticky routing 能力和轮询惩罚 |
| 限流与并发 | 10% | 429、排队、并发容量 |
| 模型纯度 | 15% | 模型身份、协议、usage 和包装信号 |
| 余额与账单可信度 | 10% | 余额风险、账单差异和 usage 可信度 |

质量状态默认规则：

| 状态 | 条件 |
|------|------|
| `production` | 质量分达标，纯度无严重问题，缓存效率可接受，余额和账单可信 |
| `watching` | 数据不足或新接入，需要继续观察 |
| `low_priority` | 可用但成本高、延迟高或轻微异常 |
| `paused` | 余额不足、错误率高、限流严重、缓存命中异常或成本异常 |
| `blocked` | 模型伪装、账单不可信、重大质量问题或人工拉黑 |

## 9. 实施阶段

### P0：内部运营看板

- 已落地：新增 `/admin/kanban` 运营看板，支持手工录入同行价格、缓存效率、供应质量快照和接入验收报告。
- 已落地：新增市场价格、缓存效率、供应质量、接入验收和看板事件表，快照只追加不覆盖历史。
- 已落地：聚合供应商倍率成本、市场价格、缓存惩罚成本、供应质量和验收状态，按模型展示市场低/中/高价、缓存调整后真实成本、建议售价、毛利率和风险等级。
- 已落地：轮询号池缓存命中率低会进入风险判断，并生成缓存风险事件。
- 已落地：自有号池和第三方供应商的缓存效率可从近 7 天 `usage_logs` 自动派生，使用真实 `cache_read_tokens`、缓存创建 token、输入/输出 token、TTFT 和总耗时计算命中率、重复输入和浪费成本。
- 已落地：供应质量可从近 7 天 `usage_logs` 与 `ops_error_logs` 自动派生，结合真实成功请求、错误请求、上游模型、余额、健康状态和并发配置生成可用率、错误率、纯度、余额风险、并发评分和质量决策。
- 已落地：供应质量归一到 `production`、`watching`、`low_priority`、`paused`、`blocked` 决策，并生成质量风险事件。
- 已落地：按质量/缓存生成验收报告时，会优先使用 usage 派生证据；没有手工快照时也能识别低缓存命中、错误日志-only 等真实风险。
- 保持约束：不自动采集复杂页面，不自动修改线上售价，不自动切流。

### P1：自动采集与价格快照

- 已部分落地：价格快照支持 `manual`、`site_catalog`、`site_discovery`、`provider_page`、`api` 来源，并可绑定站点或供应商。
- 已落地：录入新价格时自动生成价格上涨、降价和异常低价事件。
- 已落地：提供公开价格文本解析入口，可将价格页、扩展采集或渠道索引提取的文本批量结构化为市场价快照。
- 已落地：从站点目录链接和常见价格路径发现价格页候选，运营可选择候选 URL 后粘贴公开文本解析。
- 已落地：公开价格文本可识别套餐价格、套餐额度、充值赠送、最低充值门槛和倍率，并保留到价格快照。
- 已落地：公开价格文本可识别新增模型、下架模型和限时活动，并生成市场事件。
- 已落地：支持从候选公开价格 URL 抓取页面正文并自动送入价格解析器；抓取仅支持 http/https，限制响应大小，并使用解析 IP 校验降低 SSRF 风险。
- 保持约束：市场价格不能凭空真实化；没有公开价格页、站点目录、渠道采集结果或第三方供应商 API 时，仍需要运营录入或导入来源文本。
- 待实现：把渠道索引批量采集的全文抽取结果自动批量送入价格解析器。

### P2：接入前验收

- 已部分落地：新增结构化接入验收报告，覆盖连通性、模型列表、纯度检测、轻量调用、usage 计量、缓存审计、余额账单和并发限速。
- 已落地：验收结论归一到 `production`、`watching`、`low_priority`、`paused`、`blocked`；任一检查项失败默认阻断，警告或未知默认观察。
- 已落地：未通过验收会生成 `acceptance_risk` 事件，并进入模型利润风险聚合。
- 已部分落地：可从已有供应质量和缓存效率证据一键生成验收报告，缺失证据标为 `unknown`，失败证据会阻断。
- 已落地：生成验收报告时可选择排队调度证据任务，覆盖健康探针、渠道真实调用、纯度检测、usage 成本同步和余额同步；验收报告会记录调度 run id、状态和步骤数。
- 已落地：纯度检测已成为调度任务 `supplier.purity.check`，worker 会基于供应商绑定的本地账号执行纯度检测，并把 report id、模型、分数、usage token 和 token audit 摘要写入调度 step result。
- 已落地：调度 run 状态刷新后会自动识别 `kanban_acceptance` 验收任务；任务完成后将 step result 回填为供应质量快照、缓存效率快照和最终验收报告。
- 已落地：纯度检测返回的 `cached_tokens / input_tokens` 会生成缓存命中证据；路由策略未采样时保持 `unknown/none`，不把一次 usage 样本伪装成完整 sticky/轮询审计。
- 已落地：提供按 scheduler run id 手工重放回填的接口，便于自动回填失败或历史 run 补录。
- 保持约束：未通过验收的供应来源只能生成风险和建议，不自动标记为生产候选。

### P3：调度建议

- 已部分落地：基于成本、缓存效率、供应质量和市场价生成模型级风险建议与看板事件。
- 已落地：事件支持开放、已处理、已忽略状态，运营人员可在看板处理事件。
- 已落地：开放看板事件可进入动作建议生成流程，转成需审批的暂停、降权、切换、凭据复核或定价复核建议。
- 已部分落地：已审批建议可调用执行接口并写入执行回执；调研和凭据复核类记录为人工工作流成功。
- 已落地：暂停供应商建议会通过供应商状态执行器更新为 `disabled/paused`，降权建议会更新为 `monitor_only/normal`，成功或失败都会写入执行回执。
- 保持约束：切流、权重调整和充值建议仍需接入对应的路由、权重或充值专属执行器；未接入前继续返回 `unsupported` 回执。当前仓库能安全执行的范围仅限供应商状态更新和人工工作流回执。
- 保持约束：没有显式审批和专属执行器时，不直接执行线上路由调整。

## 9.2 当前事件类型

| 事件 | 触发条件 | 用途 |
|------|----------|------|
| `market_price_drop` | 同来源同模型价格下降超过阈值 | 提醒复核是否需要跟价 |
| `market_price_rise` | 同来源同模型价格上涨超过阈值 | 提醒观察市场回暖 |
| `market_price_anomaly` | 新价格显著低于同模型市场中位数 | 提醒低价可能来自低质量、限量套餐或口径差异 |
| `market_model_added` | 公开价格文本出现新增或上线模型信号 | 提醒复核是否进入报价和供应验收 |
| `market_model_removed` | 公开价格文本出现下架、停售或弃用模型信号 | 提醒复核自身是否仍在售该模型 |
| `market_promotion` | 公开价格文本出现限时、促销、折扣或赠送信号 | 提醒区分长期售价与活动价格 |
| `cache_efficiency_risk` | 缓存命中率低或状态 risky/bad | 提醒轮询号池可能造成重复输入成本 |
| `supply_quality_risk` | 供应质量决策为 low_priority/paused/blocked | 阻止低质量供应直接进入生产 |
| `acceptance_risk` | 接入验收结论为 low_priority/paused/blocked 或检查项失败 | 阻止未通过验收的供应进入生产候选 |

## 10. 验收标准

- 能录入或采集至少 3 个同行站点价格，并保留历史快照。
- 能同时展示自有号池和第三方供应商的成本、质量、余额和风险。
- 能对同一模型给出市场低价、市场中位价、真实成本、建议售价和毛利率。
- 第三方供应商接入前必须生成质量报告，失败原因可解释。
- 自有号池账号必须展示余额、额度、限流、并发、健康、缓存效率和成本指标。
- 能识别普通轮询号池导致的缓存命中率下降，并展示缓存调整后的真实成本差异。
- 价格变动、质量下降、缓存命中异常、余额不足、模型纯度异常和毛利过低都能生成后台事件。

## 11. 风险与处理

| 风险 | 处理 |
|------|------|
| 同行价格口径不一致 | 保存原始口径，同时归一化为模型维度参考价 |
| 异常低价误导决策 | 低价必须结合质量分和纯度检测，不允许单独驱动定价 |
| 第三方供应商账单不可信 | usage 计量核对失败时标记 `blocked` 或 `paused` |
| 自有号池成本难以精确 | 允许先用采购/订阅/额度成本估算，后续用实际消耗校正 |
| 轮询号池导致缓存命中率低 | 要求记录路由亲和策略，并用缓存命中审计修正真实成本和质量分 |
| 自动采集页面变化 | 采集失败只生成诊断事件，不影响历史价格快照 |
| 自动调度风险 | P0-P3 只做建议，不直接执行线上切流 |

## 12. 非目标

- 不绕过第三方登录、风控、付费墙或权限限制采集价格。
- 不承诺识别所有供应商真实上游，只基于证据链和质量报告做判断。
- 不在第一阶段开发公开前台价格榜。
- 不在第一阶段自动修改线上售价、分组倍率或账号路由。
- 不将低价供应商自动导入生产。
