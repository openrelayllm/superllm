# Release Notes

## v0.43.0 - 2026-07-16

### 供应商 Key 对齐

- 供应商详情新增“Key 对齐”页签，按分组统一核对第三方 Key、供应商绑定记录和 Sub2API 本地账号，集中展示对齐状态与待处理项。
- 支持搜索分组、Key 和本地账号，并按全部、未对齐、已对齐筛选；缺少 Key、缺少账号、绑定错误、账号停用、Key 创建中或失败等状态均独立标识。
- 对缺少 Key 和可修复绑定提供对应操作入口，已对齐关系不再显示重复操作。

### 容错与验证

- 任一对齐数据源加载失败时保留可读取结果和明确告警，同时禁用创建与修复操作，避免把不完整数据误判为真实缺失关系。
- 增加多 Key、孤立绑定、分组缺失或停用、Key 失败或停用、本地账号缺失或停用以及部分加载失败的回归测试。
- 后端执行完整 `go test -count=1 ./...` 和 `go vet ./...`；前端执行 211 个 Vitest 测试、ESLint、TypeScript 检查和生产构建。

## v0.42.5 - 2026-07-16

### 供应商工作流

- 供应商分组从大型弹窗迁移到独立详情页，集中展示分组、Key、本地 Sub2API 绑定和任务记录；旧分组深链会自动跳转到新页面。
- “创建缺失 Key”先展示完整计划，再按稳定名称逐组查询、复用或创建第三方 Key，并自动完成本地 Sub2API 账号绑定。
- 新增任务记录与分组级执行明细，明确展示创建、复用、失败、等待重试、人工处理和未执行状态。

### 容错与修复

- 第三方未提供 Key 上限或 Key 列表读取不完整时不再提前阻塞；系统显示风险提示，并在实际执行阶段继续按稳定名称查重后尝试创建。
- 第三方 Key 数量达到上限、配额无效或供应商会话需要重新验证时，当前步骤转为终态并停止剩余分组，避免无意义重试；未执行步骤保留明确原因。
- 修复分组本地 Key 上限保存成功后弹窗无法关闭，以及供应商详情页浏览器前进、后退和路由参数变化未同步的问题。

### 验证

- 后端执行完整 `go test -count=1 ./...` 和 `go vet ./...`。
- 前端执行 196 个 Vitest 测试、ESLint、TypeScript 检查和生产构建。
- 增加未知容量、截断列表继续创建、第三方上限终止剩余步骤和详情路由回归覆盖。

## v0.42.4 - 2026-07-15

### 修复

- 修复独立 `superllm` 与 `sub2api` 数据库部署下，健康探测、供应商用量判断、本地账号状态同步和路由补池错误访问单库的问题；管理状态继续写入 SuperLLM，账号、分组、凭据和用量继续访问 Sub2API。
- Sub2API 供应商 Key 容量按可用的无限制能力处理；第三方已有 Key 时读取并复用原始密钥，不重复创建第三方 Key。
- Sub2API 本地账号被删除后，可从第三方 Key 明文重新创建账号并更新原绑定；失败投影可重试接管，同名分组使用稳定后缀，Unicode 和超长分组 ID 不再被错误截断。
- Chrome 扩展复用短期供应商列表、并行读取状态，并在已知站点类型时跳过重复 API 探测；同时支持识别以文本框展示的密码输入框。

### 测试与安全

- 本地 E2E 使用独立 Sub2API 管理员测试账号，通知强制抑制，测试结束递归清理后端进程、双库 fixture、Redis 状态和调度 outbox。
- 增加双库本地账号漂移恢复/采纳、供应商 Key 复用、删除账号后重建、通知抑制和 Chrome 扩展回归测试。
- E2E 不调用真实第三方创建 Key，不发送真实飞书通知，也不建立 SuperLLM 独立用户体系。

### 验证

- 后端执行完整 `go test -count=1 ./...` 和 `go vet ./...`。
- 前端执行 196 个 Vitest 测试、ESLint、TypeScript 检查和生产构建。
- 扩展执行解析器、后台应用和站点探测测试；完整双库 E2E 及真实浏览器 `/admin/suppliers` 登录流程通过。

## v0.42.3 - 2026-07-15

### 修复

- 批量补齐 Key/账号时，第三方已有 Key 不再一律阻塞；在供应商接口可读取原始 Key 时，直接复用并创建或更新 Sub2API 本地账号和绑定。
- 第三方 Key 列表只返回掩码时，自动尝试读取 Key 详情；仍无法取得明文时保留导入投影和人工补录密钥路径，不重复创建第三方 Key。
- 开通计划和任务结果增加“复用”动作与数量，前端可直接提交仅包含复用项的计划，并正确展示“将复用”。
- Chrome 扩展支持使用临时 Token 创建供应商，不再强制要求同时提供账号和密码。

### 安全边界

- 复用的 Key 明文只在进程内传递，不写入计划、任务快照、日志或前端响应。
- 自动复用不会删除、停用或覆盖第三方 Key；掩码或无权限读取时安全阻塞并转入人工处理。

### 验证

- 后端执行完整 `go test ./...`，包含 Key 详情读取、旧 Key 复用绑定和 Token-only 供应商创建回归测试。
- 前端执行完整 Vitest、TypeScript 检查和生产构建。
- 扩展脚本执行 JavaScript 语法检查及解析器测试。

## v0.42.0 - 2026-07-12

### 架构

- SuperLLM 不再维护独立用户体系。登录、当前用户、JWT 校验、刷新令牌、管理员权限和 TOTP 均从 `SUB2API_READONLY_DATABASE_URL` 指向的现有 Sub2API 数据库读取。
- 仅允许 Sub2API 中状态为 `active` 的管理员进入 SuperLLM；本地管理员创建、开发环境密码重置和用户导入导出已删除。
- 缺少 Sub2API 只读身份源时认证失败关闭，不再回退到 SuperLLM 本地 `users` 表。

### 产品收敛

- 项目、二进制、systemd 服务、安装目录、Docker 镜像、GitHub 地址和界面品牌统一改名为 `SuperLLM`。
- 删除运营看板、代理出口与 Mihomo、邮箱自动取码、网址目录、续费提醒及对应前后端实现。
- 精简供应商操作，移除调度列、状态、会话、第三方兑换等入口；隐藏数据备份与导入导出导航，删除 Admin API Key 管理能力。
- 删除 `/api/v1/public/proxyai/*` 前台业务路由及专用 CORS/API Key 装配，账号纯度检测仅保留在管理员 API。
- 兼容迁移历史站点名 `Sub2API`、`Sub2API Admin`、`Sub2API Admin Plus`，现有安装自动显示 `SuperLLM`。

### 安装与升级

- 完善 Linux `amd64/arm64` 一键安装器，支持校验和验证、指定版本、升级、回滚、旧目录迁移及 `superllm` 管理命令。
- Web、CLI 与 Docker 安装均要求配置 Sub2API 只读数据库；Docker Compose 模板不再接受本地管理员变量。
- SuperLLM 自身读写数据库默认名统一为 `superllm`，Sub2API 身份与业务数据继续通过独立只读连接访问。
- README 增加只读 PostgreSQL 角色、首次安装、旧版迁移、升级、回滚、Docker Compose 与故障排查说明。

### 验证

- 后端执行 `go test ./...` 与管理员认证 unit tests。
- 前端执行完整 Vitest、Vue TypeScript 检查和生产构建。
- 部署脚本执行 Bash 语法检查，GoReleaser 保持 Linux-only 资产。

### 发布

- 更新版本号到 `0.42.0`。
- GitHub Release 保持 Linux-only 二进制资产：`linux_amd64`、`linux_arm64` 和 `checksums.txt`。
- tag 发布同步发布 DockerHub 与 GHCR 多架构镜像：`0.42.0`、`latest`、`0.42` 和 `0`。
- Railway 默认不自动部署；如需部署，单独启用 Release workflow 的 `deploy_railway` 或仓库变量 `RAILWAY_AUTO_DEPLOY=true`。
- 裸机 systemd 部署继续使用 GitHub Release 二进制升级；容器部署通过拉取新镜像升级。
