# Release Notes

## v0.40.0 - 2026-07-09

### 新增

- 新增本地账号运营中心，支持本地账号状态同步、漂移检测、接受本地状态、恢复托管状态和手动账号操作预览/执行。
- 新增路由补池能力，支持按本地分组预览或执行补池，并记录补池运行结果、前后容量快照和影响历史。
- 新增操作审计页面和动作执行审计快照，记录幂等键、调度来源、执行前后状态和回放标记。
- 新增供应商 Key 容量配置，支持供应商和供应商分组分别维护容量策略与容量上限。
- 新增供应商成本对账调整动作，将余额/用量成本异常纳入智能动作执行链路。

### 改进

- 调度中心补充路由容量监控和补池联动，支持低容量阈值、自动执行、冷却窗口和确认窗口配置。
- 渠道检测补充主动探测预算、预估 token 和冷却控制，调度任务可复用统一阈值。
- 本地账号候选评估纳入最近一次纯度检测结果，纯度失败会阻断补池候选，风险态会进入审查动作和前端提示。
- 本地账号候选评估纳入纯度检测快照新鲜度，7 天外检测结果会标记为过期并生成复检动作建议。
- 动作建议页新增纯度复检入口，可定位本地账号运营页并自动打开对应账号的纯度检测弹窗。
- 本地账号纯度复检会写入调度快照，复检后候选评估可立即读取最新纯度状态。
- 本地账号运营页新增最近纯度报告入口，可跳转调度中心对应 run/step 并查看结果快照。
- 本地账号运营页新增过期纯度账号选择和受控批量复检队列，逐个打开纯度检测弹窗但不自动消耗 token。
- 受控批量复检队列支持按当前页模型/能力标签圈选过期账号，便于按能力范围分批复核。
- 接入验收看板新增验收步骤矩阵，按检查项汇总通过、警告、失败和未知数量，便于定位接入阻断点。
- 模型利润看板新增目标毛利缺口和建议价相对市场中位价偏离，便于判断低倍率渠道是否仍有利润安全垫。
- 动作建议页新增超时未处理统计和行内等待时长提示，严重、警告和普通建议按不同阈值标记。
- 补充 P1/P2 阶段收口基线文档，明确 P1 主线和 P2 第一阶段关闭证据、发布前验收清单以及 P1.x/P2.x/P3 后续边界。
- 补充 P1/P2 发布前人工验收矩阵，覆盖空池发现、补池 dry-run/apply、关调度、drift、Key 配额、余额机会、纯度复检、调度来源和幂等 replay。
- 同步路由补池专题文档状态，明确调度 worker、动作建议执行历史、失败重试和成功回滚已进入 P1 收口范围。
- 同步旧重构计划和账号异步开通专项文档口径，避免历史计划被误读为当前 P1/P2 收口阻塞。
- 同步调度中心文档口径，明确本地容量巡检已接入 worker，Redis Stream 唤醒和计划编辑向导属于调度底座增强。
- 后台一键更新改为读取 `openrelayllm/sub2api-admin-plus` Release，并按 `sub2api-admin-plus_<version>_<os>_<arch>.tar.gz` 精确匹配 Linux 资产。
- 容器环境下禁用二进制自更新，提示通过拉取新镜像升级，避免容器内原地替换不可持久化。
- 本地账号候选评估纳入代理状态，代理删除、禁用、过期或不可用时阻断补池候选并生成审查动作。
- 智能动作推荐补充本地分组容量、供应商有效成本和成本快照信号，提升补池、停调度和成本修正的可解释性。
- 供应商管理增强 Key 容量状态、详情弹窗、分组容量编辑和候选供应商容量阻断提示。
- NewAPI 与 Sub2API 适配增强账号/Key/余额/用量成本同步，统一多平台可操作状态。
- 扩展采集补充供应商会话、账号和操作入口信息，提升后台自动识别与修复能力。

### 修复

- 修正本地账号操作和路由补池动作的幂等回放，避免重复点击或调度重试导致重复执行。
- 修正调度动作与人工动作的来源记录，确保审计和回滚能追踪到调度 run 与 step。
- 修正成本对账明细修复、供应商成本快照和用量成本导入的边界处理。
- 修正供应商分组、供应商 Key 和容量状态在内存仓库与 SQL 仓库中的一致性。
- 升级 Go 工具链到 `1.26.5` 并更新 AWS SDK S3/eventstream 依赖，修复 Security Scan 报告的可达漏洞。
