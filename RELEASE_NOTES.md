# Release Notes

## v0.40.0 - 2026-07-09

### 新增

- 新增本地账号运营中心，支持本地账号状态同步、漂移检测、接受本地状态、恢复托管状态和手动账号操作预览/执行。
- 新增路由补池能力，支持按本地分组预览或执行补池，并记录补池运行结果、前后容量快照和影响历史。
- 新增操作审计页面和动作执行审计快照，记录幂等键、调度来源、执行前后状态和回放标记。
- 新增供应商 Key 容量配置，支持供应商和供应商分组分别维护容量策略与容量上限。
- 新增供应商成本对账调整动作，将余额/用量成本异常纳入智能动作执行链路。

### 改进

- 调度中心补充路由容量监控和补池联动，支持低容量阈值、自动执行、冷却窗口和确认窗口配置。
- 渠道检测补充主动探测预算、预估 token 和冷却控制，调度任务可复用统一阈值。
- 本地账号候选评估纳入最近一次纯度检测结果，纯度失败会阻断补池候选，风险态会进入审查动作和前端提示。
- 本地账号候选评估纳入代理状态，代理删除、禁用、过期或不可用时阻断补池候选并生成审查动作。
- 智能动作推荐补充本地分组容量、供应商有效成本和成本快照信号，提升补池、停调度和成本修正的可解释性。
- 供应商管理增强 Key 容量状态、详情弹窗、分组容量编辑和候选供应商容量阻断提示。
- NewAPI 与 Sub2API 适配增强账号/Key/余额/用量成本同步，统一多平台可操作状态。
- 扩展采集补充供应商会话、账号和操作入口信息，提升后台自动识别与修复能力。

### 修复

- 修正本地账号操作和路由补池动作的幂等回放，避免重复点击或调度重试导致重复执行。
- 修正调度动作与人工动作的来源记录，确保审计和回滚能追踪到调度 run 与 step。
- 修正成本对账明细修复、供应商成本快照和用量成本导入的边界处理。
- 修正供应商分组、供应商 Key 和容量状态在内存仓库与 SQL 仓库中的一致性。
- 升级 Go 工具链到 `1.26.5` 并更新 AWS SDK S3/eventstream 依赖，修复 Security Scan 报告的可达漏洞。

### 测试

- 增加本地账号状态快照、漂移检测、接受/恢复、手动操作和路由补池测试。
- 增加供应商 Key 容量、分组容量、候选容量阻断和成本对账动作测试。
- 增加动作执行审计、幂等回放、调度来源和路由补池回滚测试。
- 增加前端路由、侧边栏、新运营页面和调度补池历史回归测试。
